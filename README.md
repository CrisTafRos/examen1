4.	¿Cuáles son las secciones de un proyecto bajo control de versiones con Git?
Git directory. Almacena los metadatos y la base de datos de objetos del proyecto. Se le considera la parte más importante de Git y es lo que se copia al clonar un repositorio.
Working directory. Copia de una versión del proyecto en almacenamiento para poder ser modificado o usado. Se obtiene de la base de datos comprimida.
Staging area. Es un archivo y almacena información acerca de lo que va a ir en la próxima confirmación.
5.	¿Para qué sirve una bitácora (PostgreSQL y Apache)?
Sirve para visualizar cualquier eventualidad (entre ellas warnings y errors) que hayamos tenido, con el fin de utilizarlas como una ayuda. Podemos configurarlas para saber cuánto detalle debemos tener.
